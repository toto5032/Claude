# 팀/회사 블로그 CMS — UI 기획서 & 프로토타입

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **유형** | 팀/회사 블로그 + 콘텐츠 관리 시스템 (CMS) |
| **디자인** | 심플/미니멀 (Medium 스타일) |
| **사용자** | 관리자, 작성자(팀원), 일반 방문자 |
| **핵심 기능** | 글 작성, 카테고리/태그, 댓글/좋아요, 이미지 업로드, 대시보드 |

---

## 2. 사용자 역할 (Role)

```
관리자 (Admin)     → 모든 기능 + 사용자 관리 + 대시보드
작성자 (Author)    → 글 작성/수정/삭제(본인 글) + 댓글 관리
방문자 (Visitor)   → 글 조회 + 댓글 + 좋아요
```

---

## 3. 사이트맵 (전체 페이지 구조)

```
📄 공개 페이지 (방문자)
├── / ............................ 홈 (최신글 목록)
├── /posts/{slug} ............... 글 상세 보기
├── /categories ................. 카테고리 목록
├── /categories/{slug} .......... 카테고리별 글 목록
├── /tags/{slug} ................ 태그별 글 목록
├── /search?q=키워드 ............ 검색 결과
├── /authors/{username} ......... 작성자 프로필
├── /login ...................... 로그인
└── /register ................... 회원가입

📝 작성자 페이지 (로그인 필요)
├── /write ...................... 글 작성 (에디터)
├── /write/{id}/edit ............ 글 수정
└── /my/posts ................... 내 글 관리

⚙️ 관리자 페이지
├── /admin ...................... 대시보드 (통계)
├── /admin/posts ................ 전체 글 관리
├── /admin/categories ........... 카테고리 관리
├── /admin/tags ................. 태그 관리
├── /admin/users ................ 사용자 관리
└── /admin/comments ............. 댓글 관리
```

## 5. 데이터 모델 설계

### 핵심 모델 관계도

```
User (사용자)
 ├── 1:N → Post (글)
 ├── 1:N → Comment (댓글)
 └── 1:N → Like (좋아요)

Post (글)
 ├── N:1 → User (작성자)
 ├── N:1 → Category (카테고리)
 ├── N:M → Tag (태그)          ← post_tags 중간 테이블
 ├── 1:N → Comment (댓글)
 ├── 1:N → Like (좋아요)
 └── 1:1 → Image (대표 이미지)

Category (카테고리)
 └── 1:N → Post

Tag (태그)
 └── N:M → Post

Comment (댓글)
 ├── N:1 → Post
 ├── N:1 → User
 └── N:1 → Comment (parent)   ← 대댓글 (자기참조)

Like (좋아요)
 ├── N:1 → Post
 └── N:1 → User
```

### 모델 상세

```python
# User (기존 확장)
- id, username, email, hashed_password
- display_name        # 표시 이름
- bio                 # 자기소개
- avatar_url          # 프로필 이미지
- role                # admin / author / visitor
- created_at

# Post (글)
- id, title, slug     # slug: URL용 (예: "fastapi-tutorial")
- content             # 본문 (마크다운)
- summary             # 요약문 (목록에 표시)
- thumbnail_url       # 대표 이미지
- status              # published / draft / private
- view_count          # 조회수
- reading_time        # 읽기 시간 (분)
- author_id → User
- category_id → Category
- published_at, created_at, updated_at

# Category (카테고리)
- id, name, slug, description
- post_count          # 글 수 (캐시)

# Tag (태그)
- id, name, slug

# PostTag (글-태그 연결)
- post_id → Post
- tag_id → Tag

# Comment (댓글)
- id, content
- post_id → Post
- author_id → User
- parent_id → Comment  # 대댓글
- created_at

# Like (좋아요)
- id
- post_id → Post
- user_id → User
- created_at
- UNIQUE(post_id, user_id)  # 1인 1좋아요
```

## 6. API 엔드포인트 설계

```
GET    /posts                    # 글 목록 (페이지네이션, 필터)

GET    /authors/{username}       # 작성자 프로필 + 글 목록
```

```
POST   /auth/register            # 회원가입
POST   /auth/login               # 로그인 → JWT 토큰

```

### 작성자 API (로그인 필요)

```
POST   /posts                    # 글 작성

DELETE /posts/{id}               # 글 삭제 (본인 글)
POST   /posts/{id}/like          # 좋아요 토글
POST   /posts/{id}/comments      # 댓글 작성
DELETE /comments/{id}            # 댓글 삭제
POST   /upload/image             # 이미지 업로드
```

### 관리자 API (Admin 전용)

```
GET    /admin/dashboard          # 대시보드 통계

CRUD   /admin/categories         # 카테고리 관리
CRUD   /admin/tags               # 태그 관리
CRUD   /admin/users              # 사용자 관리
DELETE /admin/comments/{id}      # 아무 댓글 삭제
```
