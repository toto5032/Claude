# 밴드 블로그 & CMS — UI 기획서 & 프로토타입

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **유형** | 음악 밴드 블로그 + 콘텐츠 관리 + 레퍼토리 관리 |
| **디자인** | 심플/미니멀 (Medium 스타일) |
| **사용자** | 관리자(밴드 리더), 멤버(밴드원), 팬(일반 방문자) |
| **핵심 기능** | 블로그, 레퍼토리/후보곡 투표, 공연 일정, 디스코그래피, 멤버 소개, 갤러리, 대시보드 |

---

## 2. 사용자 역할 (Role)

```
관리자 (Admin)   → 모든 기능 + 곡 상태 변경 + 사용자 관리 + 대시보드
멤버 (Member)    → 글 작성 + 곡 추천/투표/메모 + 댓글 관리
팬 (Fan)         → 글 조회 + 곡 추천/팬투표 + 댓글 + 좋아요
```

---

## 3. 사이트맵 (전체 페이지 구조)

```
📄 공개 페이지 (팬/방문자)
├── / ............................ 홈 (밴드 히어로 + 최신 소식)
├── /posts/{slug} ............... 글 상세 보기
├── /categories ................. 카테고리 목록
├── /categories/{slug} .......... 카테고리별 글 목록
├── /tags/{slug} ................ 태그별 글 목록
├── /search?q=키워드 ............ 검색 결과
├── /members .................... 밴드 멤버 소개
├── /shows ...................... 공연 일정 (다가오는 공연 + 지난 공연)
├── /discography ................ 앨범 목록
├── /discography/{slug} ......... 앨범 상세 (트랙 + 플레이어)
├── /repertoire ................. 레퍼토리 목록 (확정/연습중/후보)
├── /repertoire/{id} ............ 곡 상세 (YouTube + 투표 + 메모)
├── /repertoire/suggest ......... 곡 추천하기 (로그인 필요)
├── /gallery .................... 사진/영상 갤러리
├── /login ...................... 로그인
└── /register ................... 회원가입 (팬)

🎵 멤버 페이지 (밴드원 전용)
├── /write ...................... 글 작성 (에디터)
├── /write/{id}/edit ............ 글 수정
├── /my/posts ................... 내 글 관리
└── /repertoire/{id}/vote ....... 멤버 투표

⚙️ 관리자 페이지 (밴드 리더)
├── /admin ...................... 대시보드 (통계)
├── /admin/posts ................ 전체 글 관리
├── /admin/categories ........... 카테고리 관리
├── /admin/tags ................. 태그 관리
├── /admin/repertoire ........... 레퍼토리 관리 (상태 변경)
├── /admin/shows ................ 공연 일정 관리
├── /admin/albums ............... 앨범/곡 관리
├── /admin/members .............. 멤버 정보 관리
├── /admin/gallery .............. 갤러리 관리
├── /admin/users ................ 사용자 관리
└── /admin/comments ............. 댓글 관리
```

## 5. 데이터 모델 설계

### 전체 모델 관계도

```
User (사용자: 관리자/멤버/팬)
 ├── 1:N → Post (블로그 글)
 ├── 1:N → Comment (댓글)
 ├── 1:N → Like (글 좋아요)
 ├── 1:N → Song (곡 추천)
 ├── 1:N → SongVote (멤버 투표)
 ├── 1:N → SongFanVote (팬 투표)
 └── 1:N → SongComment (곡 메모)

Post (블로그 글)
 ├── N:1 → User (작성자)
 ├── N:1 → Category (카테고리)
 ├── N:M → Tag (태그)            ← post_tags 중간 테이블
 ├── 1:N → Comment (댓글)
 └── 1:N → Like (좋아요)

Category (카테고리)
 └── 1:N → Post

Tag (태그)
 └── N:M → Post

Comment (댓글)
 ├── N:1 → Post
 ├── N:1 → User
 └── N:1 → Comment (parent)     ← 대댓글 (자기참조)

Like (좋아요)
 ├── N:1 → Post
 └── N:1 → User
 └── UNIQUE(post_id, user_id)

Member (밴드 멤버 프로필)
 - 별도 관리 (User와 1:1 연결 가능)

Album (앨범)
 └── 1:N → Track (곡)

Track (트랙)
 └── N:1 → Album

Show (공연)
 └── 1:N → SetlistItem (세트리스트)

Song (레퍼토리 곡)
 ├── N:1 → User (추천자)
 ├── 1:N → SongVote (멤버 투표)
 ├── 1:N → SongFanVote (팬 투표)
 ├── 1:N → SongComment (곡 메모)
 └── 1:N → SetlistItem (세트리스트)

SetlistItem (공연 세트리스트)
 ├── N:1 → Show
 └── N:1 → Song
```

### 모델 상세

```python
# ── 사용자 ──

# User (기존 확장)
- id, username, email, hashed_password
- display_name          # 표시 이름
- bio                   # 자기소개
- avatar_url            # 프로필 이미지
- role                  # admin / member / fan
- created_at

# ── 블로그 ──

# Post (글)
- id, title, slug
- content               # 본문 (마크다운)
- summary               # 요약문
- thumbnail_url         # 대표 이미지
- status                # published / draft / private
- view_count            # 조회수
- reading_time          # 읽기 시간 (분)
- author_id → User
- category_id → Category
- published_at, created_at, updated_at

# Category (카테고리)
- id, name, slug, description
- post_count            # 글 수 (캐시)

# Tag (태그)
- id, name, slug

# PostTag (글-태그 연결)
- post_id → Post
- tag_id → Tag

# Comment (댓글)
- id, content
- post_id → Post
- author_id → User
- parent_id → Comment   # 대댓글
- created_at

# Like (좋아요)
- id
- post_id → Post
- user_id → User
- created_at
- UNIQUE(post_id, user_id)

# ── 밴드 정보 ──

# Member (멤버 소개)
- id, name, role_in_band  # 기타, 보컬, 드럼 등
- photo_url, bio
- sns_links             # JSON: {"instagram": "...", "youtube": "..."}
- sort_order            # 표시 순서
- user_id → User        # 선택: 로그인 계정 연결

# Album (앨범)
- id, title, slug
- cover_url             # 앨범 커버 이미지
- release_date
- description
- streaming_links       # JSON: {"spotify": "...", "apple_music": "..."}

# Track (트랙)
- id, title
- track_number          # 트랙 번호
- duration              # 재생 시간 (초)
- lyrics                # 가사
- audio_url             # 오디오 파일 또는 스트리밍 링크
- album_id → Album

# ── 공연 ──

# Show (공연)
- id, title
- venue, address        # 공연장, 주소
- show_date, show_time
- ticket_url            # 티켓 예매 링크
- ticket_price          # 가격
- poster_url            # 포스터 이미지
- status                # upcoming / past / cancelled
- description

# ── 레퍼토리 ──

# Song (레퍼토리 곡)
- id, title, artist
- youtube_url           # YouTube 링크
- youtube_video_id      # 영상 ID (임베드용, URL에서 자동 추출)
- thumbnail_url         # YouTube 썸네일 (자동)
- genre                 # 장르
- status                # candidate → practicing → ready → archived
- reason                # 추천 이유
- suggested_by → User   # 추천한 사람 (멤버 또는 팬)
- created_at, updated_at

# SongVote (멤버 투표)
- id
- song_id → Song
- user_id → User        # 멤버만 가능
- vote_type             # up / down
- created_at
- UNIQUE(song_id, user_id)

# SongFanVote (팬 투표)
- id
- song_id → Song
- user_id → User        # 팬 가능
- created_at
- UNIQUE(song_id, user_id)

# SongComment (곡 메모)
- id, content
- song_id → Song
- user_id → User
- created_at

# SetlistItem (공연 세트리스트)
- id
- show_id → Show
- song_id → Song
- play_order            # 연주 순서

# ── 갤러리 ──

# GalleryItem (갤러리)
- id, title
- media_type            # photo / video
- media_url             # 이미지/영상 URL
- thumbnail_url
- category              # concert / practice / offstage
- show_id → Show        # 선택: 공연 연결
- created_at
```

```
# 블로그

GET    /members                      # 멤버 소개

# 레퍼토리
GET    /repertoire                   # 레퍼토리 목록 (상태별 필터)

# 갤러리
GET    /gallery                      # 갤러리 목록
```

```
POST   /auth/register                # 회원가입 (팬)

POST   /auth/logout                  # 로그아웃
```

```
# 블로그

DELETE /comments/{id}                # 댓글 삭제 (본인)

```

POST   /repertoire/{id}/vote         # 멤버 투표
POST   /repertoire/{id}/comments     # 곡 메모 작성

## 7. 개발 순서 (권장)

```
Phase 1 — 기반 구축 & 밴드 기본 정보
  ① User 모델 확장 (role: admin/member/fan, display_name, bio, avatar_url)
  ② Member 모델 + 멤버 소개 페이지
  ③ 홈페이지 리디자인 (밴드 히어로 + 네비게이션)
  ④ 기본 레이아웃/CSS (base.html, 네비게이션, 푸터)

Phase 2 — 레퍼토리 & YouTube 연동 (핵심 기능)
  ⑤ Song 모델 + YouTube URL 파싱 (video_id, 썸네일 자동 추출)
  ⑥ 레퍼토리 목록 페이지 (상태별 필터 탭)
  ⑦ 곡 상세 페이지 (YouTube 임베드 플레이어)
  ⑧ 곡 추천하기 (팬/멤버 모두 가능, YouTube URL 입력)
  ⑨ SongVote (멤버 투표) + SongFanVote (팬 투표)
  ⑩ SongComment (곡 메모: 키 변경, 어레인지 의견 등)
  ⑪ 관리자 레퍼토리 관리 (상태 변경: 후보→연습중→확정→보관)

Phase 3 — 공연 관리
  ⑫ Show 모델 + 공연 일정 페이지 (다가오는/지난 공연)
  ⑬ SetlistItem 모델 + 공연 세트리스트 (레퍼토리에서 곡 배정)
  ⑭ 홈페이지에 다가오는 공연 표시
  ⑮ 관리자 공연 관리 페이지

Phase 4 — 블로그 & 소통
  ⑯ Post 모델 (title, slug, content, summary, status)
  ⑰ Category + Tag 모델 + 관계 설정
  ⑱ 홈페이지 최신 소식 섹션
  ⑲ 글 상세 페이지
  ⑳ 글 작성/수정 에디터 (멤버 전용)
  ㉑ Comment (댓글, 대댓글) + Like (좋아요)
  ㉒ 검색 기능

Phase 5 — 음악 & 미디어
  ㉓ Album + Track 모델 + 디스코그래피 페이지
  ㉔ 앨범 상세 (트랙 목록, 가사, 스트리밍 링크)
  ㉕ GalleryItem 모델 + 갤러리 페이지
  ㉖ 이미지/파일 업로드 기능

Phase 6 — 관리자 & 마무리
  ㉗ 관리자 대시보드 (통계: 글 수, 레퍼토리 현황, 팬 수, 다음 공연)
  ㉘ 관리자 전체 관리 페이지 (글/댓글/사용자/갤러리)
  ㉙ 임시저장 / 발행 상태 관리
  ㉚ 조회수 / 읽기 시간 자동 계산
  ㉛ 반응형 디자인 (모바일/태블릿) 점검
```

## 8. YouTube 연동 방식

```
곡 추천 시:
1. YouTube URL 입력
   예: https://youtube.com/watch?v=fJ9rUzIMcZQ
       https://youtu.be/fJ9rUzIMcZQ

2. video_id 자동 추출
   → "fJ9rUzIMcZQ"

3. 썸네일 자동 설정 (API 키 불필요)
   → https://img.youtube.com/vi/fJ9rUzIMcZQ/maxresdefault.jpg

4. 상세 페이지에서 임베드 플레이어 표시
   → <iframe src="https://youtube.com/embed/fJ9rUzIMcZQ">

향후 확장 (선택):
- YouTube Data API v3 연동 → 곡 제목, 아티스트, 재생시간 자동 입력
- YouTube 검색 기능 → URL 직접 입력 대신 검색으로 곡 찾기
```
